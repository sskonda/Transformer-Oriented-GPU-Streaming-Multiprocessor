# Future Work

## Tensor Datapath

- Implement and verify iterative or systolic mode
- Add architectural storage for the full result matrix
- Evaluate pipeline depth, initiation interval, area, and timing after synthesis
- Explore structured sparsity only with an explicit encoding and golden model

## Scheduling And Dependencies

- Add starvation control to greedy policies
- Predict prefetch completion and shared-memory conflicts
- Measure operand forwarding on dependency-heavy workloads
- Add more realistic multi-warp overlap workloads

## Memory System

- Separate A and B tile operands
- Add multiple outstanding global-memory transactions
- Add coalescing, cache, or NoC-facing protocol adapters
- Evaluate bank count and mapping sensitivity

## Software And Workloads

- Lower MLP and attention operations into tiled WarpForge programs
- Add register allocation and tile lifetime management
- Add a runtime loader and architectural result-memory interface
- Support larger tiled GEMM loops

## Verification

- Run native constrained randomization, assertions, and functional coverage in
  a licensed simulator
- Add dedicated queue-level, state-transition, and conflict covergroups
- Add formal protocol and deadlock properties
- Add randomized memory latency and backpressure to the direct Verilator flow
- Run many-seed regressions in CI when simulator licensing permits

## Implementation Results

- Run FPGA synthesis and timing analysis
- Report device, tool version, constraints, resource utilization, and Fmax
- Compare tree, pipelined-tree, and future systolic implementations

No synthesis or FPGA result should be added until it is generated by an
actual tool run with reproducible settings.
